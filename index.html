HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • Sematic elements make the HTML easier to read by other developers
  • There are opening tags and closing tags to allow the browser to understand how to treat the content.
  • Elements are the entire sections and tags are the content inside the sections
  • Headers need to be numbered in order in each section to keep them in the correct placement
  • Parent elements hold all the child elements for example: Main "Parent Element" Holds h2, ul, li "Child Elements"

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • There are three ways to style a webpage, inline CSS, internal CSS style sheet, and external CSS style sheet.
  • Inline CSS, is when style an element directly into the html file.
  • Internal style sheet contains CSS rules for a webpage in the head section of the HTML file.
  • External style sheet is most common, which allows developers to keep all CSS rules in a seperate file.
  • When using an external style sheet you will need to "link" in the "head" of the html file.
  • Declarations contain two components, property and value of that property. Such as color:blue.
  • Margin(property) is the space outside of the element, while Padding(property) is the space inside the element. Each will need a value.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -a: to add the starter code the the current working branch
  • git commit -m "add starter code": to apply a message that contains what changes were made
  • git pull orgin-main: to double check if our local branch is in sync with the base branch in GitHub
  • git push orgin branch-name: pushes changes to the remote branch
  • Go to GitHub and review pull request and create for others to review before merging into the base branch

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript allows the user to interact with the website
  • Variables contain strings that include all the topics or sections
  • A method such as console.log is created to output a message to the web console by adding an argument.
  • To interact with the webpage you can open up Chrome DevTools and review the console tab to debug issues
  • console.log is used to test code
  • Conditional statements and loops are used to control the order in which the code is executed
  • Array's are used to test values on a large amount of data
  • Index is a location within an Array
  • Commenting out code makes it inactive without deleting, in case it is needed later
  • Function is a simple set of instructions that tells the computer how to perform a task
